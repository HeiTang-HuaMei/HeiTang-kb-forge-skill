# Capability Implementation Status

Status: `capability_inventory_routing_completed_needs_owner_review`

This registry is an execution ledger, not a roadmap essay. Every capability below is routed as landed, reserved, partial, missing, or blocked. `needs_owner_review` means the evidence exists but the owner has not accepted it as final. It does not mean fully passed, release ready, production ready, or industrial acceptance passed.

## Status Vocabulary

- `implemented`: real repository code or runtime behavior exists.
- `metadata_reserved`: fields, manifests, statuses, or artifact/event hooks exist, but the runtime ability is not implemented.
- `partial`: some behavior or evidence exists, but required acceptance scope is incomplete.
- `display_only`: visible UI or documentation exists without enough runtime behavior.
- `mock_only`: mocked or synthetic behavior exists and is not accepted as product capability.
- `not_implemented`: no accepted implementation evidence exists.
- `blocked`: cannot proceed without missing dependency or owner decision.
