# 产品定位

当前产品事实以 [docs/product](product/) 下三份 v3 文档为唯一基线。本文只解释产品定位，不替代 PRD 或验收矩阵。

## 定位

HeiTang Knowledge Workbench 是本地优先的知识资产工作台，不是文件管理器，不是单纯 RAG 问答壳，也不是默认联网的 SaaS 平台。

它的目标是把资料转成可管理、可检索、可验证、可编排、可导出的知识资产：

```text
文档库 → 知识库 → 索引层 → RAG → 编排层 → 文档/Skill/Agent/A2A
```

## 做什么

- 把文件、文件夹、网页链接或外部 Skill 资料解析成文档库内容。
- 从文档库选择内容，构建一个或多个知识库。
- 为知识库生成关键词索引、向量索引、元数据索引、引用索引和记忆索引。
- 通过 RAG 与验证层提供召回、重排、引用、冲突检测、人工纠偏和验证报告。
- 通过编排层生成文档、Skill、单 Agent 和 A2A 协作产物。
- 在产物中心、审计中心和导出流程中保留来源、配置、版本和复验依据。

## 不做什么

- 不把文档库写成原始文件夹或简单文件列表。
- 不把知识库写成没有索引、引用和质量报告的文件堆。
- 不把 RAG 写成无 KB、无引用、无验证的直接回答。
- 不把 OKF 写成当前已实现 runtime、一级导航或用户主链路。
- 不把登记项目写成已接入；登记项目只能是 `reference_only`、`needs_verification` 或经单独验收后的候选状态。
- 不把 UI handoff 写成 Campaign 4 UI 完成。
- 不把 Bridge handoff 写成 Campaign 5 Bridge 完成。
- 不默认上传用户资料，不保存真实 API key，不绕登录、验证码、付费墙或平台限制。
- 不开放 arbitrary shell、Computer Use 或未授权插件市场。

## 四类传统产物与 v3 关系

历史 Core 文档中的四类产物仍作为产物层术语保留：

1. Knowledge Package
2. Document Outputs：Markdown / DOCX / PDF / PPTX
3. Skill Outputs：Skill Template / Skill Suite
4. Agent Creation Package

在 v3 口径下，它们统一归入“编排层 → 文档/Skill/Agent/A2A → 产物中心”链路。Agent Creation Package 不等于 Agent Runtime ready；Agent Runtime ready 只能作为历史边界词出现，不得写成当前已完成事实。

## 安全与隐私边界

- 默认本地优先。
- 外部 provider 必须由用户显式配置。
- 网络、Embedding、向量库、Redis 和导出器均按配置启用。
- `reference_only` 项目只能作为方法或架构参考。
- Redis / Vector DB 是 future target 或配置项，不得直接写成当前 runtime 已接入。
- 不把 Redis / Vector DB future target 写成当前 runtime。

## 历史兼容术语

以下内容保留给旧 Core 审计和公共文档测试追溯，不替代 v3 三基线：

- Memory Separation / Knowledge Lifecycle
- Evidence Map / Source Trace
- Retrieval / Verification
- Workspace Partition / KB Access Scope
- External Source Memory & Verification
- Document generation
- Skill generation
- Agent package generation

Document Outputs：Markdown / DOCX / PDF / PPTX 是正式产品能力，不是审计报告副产物，也不被 Skill Outputs 覆盖。
