# 发布流程

本文件只补充文档基线 gate，不替代既有工程 release 文档。

## Release Gate

发布前必须确认：

1. PRD / 架构 / 验收三基线一致。
2. 当前文档只引用、解释或落地三基线。
3. 主链路统一为“文档库 → 知识库 → 索引层 → RAG → 编排层 → 文档/Skill/Agent/A2A”。
4. 登记项目未被写成已接入。
5. OKF 未被写成当前 runtime、一级页面或用户主链路。
6. 旧 rc / campaign / audit 文档只作历史追溯。

## 禁止项

- 不创建 GitHub Release。
- 不打 stable v4.3.0。
- 不新增 runtime 功能。
- 不开放 arbitrary shell、Computer Use 或未授权插件市场。
