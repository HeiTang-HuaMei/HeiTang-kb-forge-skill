# 外部运行时参考队列

本队列只记录登记项目的分流结果，不代表外部项目已经进入当前
runtime。普通用户界面不得暴露外部项目、热插拔项目、插件市场、
Gate/Campaign 或 Core 操作概念。

## 当前规则

- 先判断登记项目是否值得吸收，再决定进入 Provider、模板资产或架构参考分流。
- 值得吸收的架构参考必须并行交付 contract、schema、runtime 边界、UI 信息架构边界、测试 gate、审计模型、降级策略或 Provider 分类/加载规则。
- 不值得吸收的登记项目必须写明拒绝原因，不得长期挂在参考队列。
- 无法立即吸收的登记项目只能使用 `deferred_with_blocker`，且必须有明确 blocker。
- Provider、Agent runtime、Computer Use、shell 或外部服务能力必须单独走安全和验收 gate。
- OKF 不在本队列实现 runtime；Stage3 只处理 Provider 化配置、加载边界、审计和回滚。

## 分流状态

| 状态 | 含义 |
| --- | --- |
| `capability_provider` | 能增强 Parser/OCR、Embedding/Vector、Search/Retrieval、Exporter、Agent 记忆/工具、A2A 导出或治理评测能力，且可配置、测试、审计、回滚。 |
| `template_asset` | 提供提示词、方法论、模板、写作框架或 Skill/Agent 范式，不加载外部 runtime。 |
| `absorbed_into_architecture` | 参考已转化为架构合同、schema、边界、gate、审计、降级或加载规则。 |
| `rejected_no_architecture_gain` | 对 v3 主链路无明确增益、成本高于收益、与架构冲突或已被现有能力覆盖。 |
| `deferred_with_blocker` | 有吸收价值，但存在明确 blocker；不得无限期保留。 |

## 最小字段

| 字段 | 含义 |
| --- | --- |
| 项目 | 外部项目或标准名称 |
| 分流 | `capability_provider` / `template_asset` / 架构参考状态 |
| 吸收价值 | 能补的架构短板或明确无增益原因 |
| 交付结果 | Provider 配置、模板 manifest、架构合同、测试 gate、审计或拒绝记录 |
| 当前边界 | runtime、网络、权限、secret、UI 暴露和回滚边界 |
| 下一步 | 验证、吸收、拒绝或解除 blocker |

## 当前结论

登记项目不能用学习笔记替代交付。Stage3 的有效交付必须让产品能力、
架构边界、测试 gate 或审计回滚模型变得更清晰；否则必须明确拒绝。
