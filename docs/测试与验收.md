# 测试与验收

## 当前主分支验收目标

main 分支应表现为干净的 v4.2 产品基线，而不是历史审计工作区。

## 必须验证

- 根目录可见文件数量受控。
- 根目录 JSON 只允许 `skill.json`。
- 不提交 `artifacts/audits/current_run` 或 `artifacts/audits/latest`。
- 不保留 pre-v4.2 根目录旧 RC / gate / absorption JSON。
- 人读 docs 使用中文文件名。
- Python 包、tests、scripts、GitHub workflows 等工程关键路径保持英文。
- 运行时能力通过测试在临时目录中生成证据，不依赖 main 中旧审计堆。
- optional OCR / advanced parser 是 dependency-gated，不是默认 bundled runtime。

## 文档输出治理

文档输出分为：

- Product docs
- Governance rules
- Contracts and schemas
- Audit evidence
- Runtime logs and caches

当前 public main 不提交运行时审计堆。`artifacts/audits/latest/`、`artifacts/audits/daily/`、`progress_events.jsonl`、运行时 cache 与本地依赖修复目录均应被 `.gitignore` 拦截。

运行时证据应在本地临时目录生成：

- `run_manifest.json`
- `run_summary.md`
- 结构化 JSON report

旧保留规则只作为历史背景：newest 3 runs、Keep 7 days、Keep 3 days unless promoted 均不表示 public main 需要保留这些文件。当前 v4.2 reset 的默认策略是 not committed by default。

## 推荐命令

```powershell
python -m pytest tests/test_v4_2_public_repository_reset.py -q
python -m pytest -q
python -m json.tool .\skill.json
git diff --check
```
