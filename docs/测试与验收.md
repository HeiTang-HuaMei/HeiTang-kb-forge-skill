# 测试与验收

当前验收以 [功能评审 / 验收矩阵 v3](product/FEATURE_ACCEPTANCE_MATRIX_V3_2026-06-19.md) 为准。本文只说明 Core 仓库的 gate 落地。

## v3 验收主线

验收时按以下链路复验：

```text
文档库 → 知识库 → 索引层 → RAG → 编排层 → 文档/Skill/Agent/A2A
```

核心模块目标为 L3：链路真实可用、产物可追溯、重启一致、失败原因可见、下游引用不越权。

## 必验模块

- 工作区：创建、切换、删除保护、持久化、资源隔离。
- 导入与解析：真实文件 / 文件夹导入、document_id、正文、chunks、解析报告、失败重试。
- 文档库：多文档持久化、搜索、预览、chunks、关联 KB、删除影响提示。
- 标准知识包 / OKF：只验证候选层边界，不验证 runtime。
- 知识库：多 KB、来源追溯、构建配置、index metadata、quality report、版本记录。
- 索引层：关键词、向量、元数据、引用、记忆索引及同步删除 / 重建。
- RAG 与验证：单 KB / 多 KB、引用、覆盖率、冲突检测、人工纠偏、验证报告。
- 编排层：文档、Skill、Agent、A2A 的来源、配置、Provider、审计记录。
- 产物中心：查看、删除、导出一致性。

## Fast Gate

迁移阶段优先运行 Fast Gate：

```powershell
python -m pytest tests/test_final_docs_structure.py tests/test_final_docs_truthfulness.py tests/test_release_checklist_docs.py -q
python -m json.tool .\skill.json
git diff --check
```

涉及 UI runtime 的仓库另行运行 Flutter analyze 和目标测试。Full Gate 仅在明确需要或远端 CI 中执行。

## Release Check 必备输入

Release Check 必须覆盖：

- `python -m pytest`
- Quickstart
- Release Check
- `git diff --check`
- PRD / 架构 / 验收三基线一致性
- overclaim scan
- no-secret scan
- OKF runtime scan

## 历史兼容边界

旧 Core 测试仍可能检查以下历史词：Knowledge Package、Document Outputs、Markdown / DOCX / PDF / PPTX、Skill Template、Skill Suite、Agent Creation Package、optional OCR / advanced parser 是 dependency-gated。这些词只作为现有 Core 产物和依赖门控说明保留，不得覆盖 v3 验收矩阵。
