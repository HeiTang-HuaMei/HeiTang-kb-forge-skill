# Post-v4 External Roadmap

Ordering rule: function strengthening first, experience second, ecosystem later.

Additional rules:

1. Absorb at most two S/A directions per version.
2. Do not implement multiple S-level large systems in one version.
3. External projects are capability-fusion references, not full-copy targets.
4. Provider, network, and external runtime work requires explicit user configuration.
5. This roadmap does not modify `v4.0.0` or start P2.2 inside the v4.1.0 P2.1 release closure.

## Suggested Route

| Phase | Focus | Boundary |
| --- | --- | --- |
| P2.1 | External Project Verification Baseline + Parser/OCR Multi-Backend Integration | Verification baseline closed; Docling / PaddleOCR / Unstructured optional runtime adapters release-hardened for v4.1.0 |
| P2.2 | Skill Governance + Book-to-Skill Deepening | andrej-karpathy-skills, skill-prompt-generator, Book-to-Skill / Software-to-Manual-to-Skill |
| P2.3 | Parser/OCR Backend Hardening | MinerU, Marker, Surya, OpenDataLoader; acceptance hardening for optional runtime adapters |
| P2.4 | Living Knowledge Base / Memory Lifecycle | LLM Wiki v2; memory lifecycle / confidence / forgetting / decay |
| P2.5 | Auto Wiki + RAG Evaluation | WeKnora; LlamaIndex / RAGAS / DeepEval |
| P2.6 | Industry Template Library + AIGC Book Pipeline | ai-marketing-skills, ai-money-maker-handbook, AIGC Book Content Pipeline |
| P2.7 | n8n Workflow Export Adapter | n8n workflow export and webhook templates; no bundled runtime |
| P2.8 | External Retrieval Provider | AnySearchSkill, last30days-skill; provider_required / network_required |
| P2.9 | Multimodal / Short Drama / Video Skill Pipeline | MMSkills, Jellyfish, story-flicks, seedance2-skill |
| P3.x | Ecosystem & Automation Layer | deeper n8n integration, external automation provider registry, workflow canvas, BYO cloud / sync |
| P4.x | SaaS / Team / Commercialization | team workspace, permission model, hosted control plane, plugin ecosystem, commercial packaging |
