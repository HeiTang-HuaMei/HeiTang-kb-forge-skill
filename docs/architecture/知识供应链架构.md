# 知识供应链架构

知识供应链把资料转成可验证、可复用、可编排的知识资产。

## 标准链路

```text
资料导入
  → 解析 / 清洗 / 结构化
  → 文档库
  → 标准知识包 / OKF 候选层
  → 知识库
  → 索引层
  → RAG 与验证层
  → 编排层
  → 文档 / Skill / Agent / A2A
```

## UI 应展示的证据

- 导入任务和失败原因。
- 文档正文、chunks 和结构统计。
- KB manifest、quality report 和版本。
- index metadata。
- RAG 命中 chunks、引用、覆盖率和验证报告。
- 文档、Skill、Agent、A2A 的编排记录。
- 产物中心和审计中心记录。

## 标准知识包 / OKF 候选层

标准知识包用于版本化、流通和追溯。OKF 只作为候选标准描述；OKF Bundle 导出、OKF → KB 构建和 OKF + Agent 间接使用属于后续验收，不得写作当前 runtime。
