# 系统架构

当前系统架构以 [产品架构 v3](../product/PRODUCT_ARCHITECTURE_V3_2026-06-19.md) 为准。

## 分层

```text
L0 工作区层 Workspace
L1 资料接入层 Import Source
L2 文档库层 Document Library
L3 标准知识包 / OKF 候选层
L4 知识库层 Knowledge Base
L5 索引层 Index Layer
L6 RAG 与验证层 Retrieval / RAG / Validation
L7 编排层 Orchestration
L8 产物层 Artifacts
L9 治理配置层 Governance / Settings
```

## UI 职责

Workbench UI 负责把每层能力呈现为可操作状态：

- 输入和状态必须来自真实 runtime 或明确的 disabled boundary。
- 产物路径、审计记录和导出记录必须可见。
- 失败必须显示原因。
- 重启后状态必须一致。
- 不把 mock、历史报告或计划文档显示成当前已完成能力。

## Provider 层

Provider 包括 LLM、Embedding、Parser、OCR、Redis、向量库、Search 和 Exporter。UI 可以显示配置与测试结果，但未配置或失败时不得假成功。

## OKF 边界

OKF 只属于标准知识包候选层。当前不新增 OKF runtime，不新增 OKF 一级导航，不改变用户主链路。
