# 发布流程

本文件属于工程协作 / release gate 文档，不替代 PRD。当前发布判断必须先满足 v3 三基线一致：

- [产品架构 v3](product/PRODUCT_ARCHITECTURE_V3_2026-06-19.md)
- [PRD v3](product/PRD_V3_2026-06-19.md)
- [功能评审 / 验收矩阵 v3](product/FEATURE_ACCEPTANCE_MATRIX_V3_2026-06-19.md)

## Release Gate

发布前必须确认：

1. PRD / 架构 / 验收三基线一致。
2. 当前文档只引用、解释或落地三基线，不另立产品事实。
3. 主链路统一为“文档库 → 知识库 → 索引层 → RAG → 编排层 → 文档/Skill/Agent/A2A”。
4. 登记项目只能写为 `reference_only` / `needs_verification` / 已单独核验的候选状态，不得写成已接入。
5. OKF 只写成标准知识包候选层，不写成 runtime、一级导航或当前用户主链路。
6. 旧 rc / campaign / 验收报告只作历史追溯。
