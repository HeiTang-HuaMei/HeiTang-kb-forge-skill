# 路线图

当前路线图按 [PRD v3](product/PRD_V3_2026-06-19.md) 和 [功能评审 / 验收矩阵 v3](product/FEATURE_ACCEPTANCE_MATRIX_V3_2026-06-19.md) 的 P0 / P1 / P2 组织。旧 Campaign 只作历史追溯，不再作为当前产品路线事实源。

## P0 当前主链路

P0 聚焦用户能完成真实闭环：

- 工作区创建、切换、删除、持久化和最小配置。
- 文件、文件夹、网页链接、外部 Skill 的导入与解析。
- 文档库持久化、搜索、预览、chunks 查看、关联 KB、删除影响提示。
- 从文档库创建一个或多个知识库，生成 manifest、chunks、quality report 和 index metadata。
- 构建关键词、元数据、引用等基础索引。
- 单 KB / 多 KB 检索、引用证据、验证报告和人工纠偏。
- 文档生成、Skill 生成、单 Agent 创建、A2A 协作产物。
- 产物中心、审计中心、导出和重启一致性。

## P1 增强闭环

P1 强化工业级可用性：

- 多 KB 复制、合并、拆分、增量更新、全量重建、版本比较和回滚。
- 向量索引、记忆索引和外部向量库的配置化启用。
- 查询改写、重排、覆盖率指标、冲突检测、外部事实验证。
- Skill 草稿编辑、验证、导出、版本管理和二次融合。
- Agent 工作区权限审计、KB/Skill 绑定审计、对话历史和导出一致性。
- A2A 多轮协作、冲突检测、审计和报告导出。

## P2 标准化与规模化

P2 作为后续目标：

- 标准知识包 Bundle 导出。
- OKF Bundle 导出。
- OKF → 知识库构建。
- OKF + Skill 个性化。
- OKF + Agent 通过 KB / Skill 间接使用来源、标签、版本和引用。
- 更大规模文档库、多租户治理和高级编排。
