## 后续战役边界

- Campaign 4 是产品 UI 工作台，不是 Bridge completion、不是 Agent Runtime completion、不是 EXE packaging。
- Campaign 4 顶层导航不超过七个入口，Agent package spec 不得显示成 runtime ready，Agent runtime 不得显示成 ready，Multi-Agent spec 不得显示成 executable。
- TasteSkill 或 Product Design Plugin 不属于 Campaign 4 base scope。
- Campaign 5 是链路级 Local Core Bridge，覆盖 `import_to_kb_flow`、`external_source_verification_flow`、`agent_package_generation_flow`、`agent_output_verification_flow` 等用户任务链路，但不得允许 arbitrary shell execution。
- Campaign 5 可复用 `generate-agent` 与 `validate-agent-package` 检查 Agent Creation Package 产物；`run-agent-task`、`agent-memory-read` 属于 Campaign 6，不得进入 Campaign 5 Bridge allowlist。
- Campaign 6 才处理 Agent Foundation：创建、编辑、复制、版本管理、简易/复杂模式、知识库绑定、多 Skill 绑定、基础模型/工具/权限配置、工作分区声明、验证、保存、预览和 Agent package 导出。Agent package 只是导出产物，不是一级功能区。
- Campaign 6：Agent Runtime / Memory 才能验收 `simple_single_agent_mode_runtime`、`advanced_single_agent_mode_runtime`、Agent memory isolation、Redis / Vector DB memory runtime。Redis config 或 Vector DB config alone is not Agent Runtime & Memory acceptance。
- Campaign 6 Entry Gate 必须冻结 simple-mode field matrix、advanced-mode field matrix、Agent schema，以及每个可见动作的 capability classification：`enabled_real`、`disabled_boundary`、`display_only` 或 `omitted`。
- Campaign 6 删除行为只允许 archive 或 recoverable soft deletion，不得实现不可逆物理删除。
- Campaign 7：配置系统才处理配置系统工程化：Profile 文件化、配置来源优先级、加载、合并、迁移、向后兼容、Secret 安全注入、诊断、Runtime availability check 和批量复用 Agent Profile。不得在 Campaign 7 才首次加入模型、工具、权限、工作分区或模式核心字段。
- Campaign 7/8 之前不得把 `check-agent-runtime`、`check-agent-memory-backend`、`check-opencli` 的存在当成验收通过；它们只能作为后续 Gate 的验证命令。
- Campaign 8：Full Testing / Review 才处理 Full Testing / Full Review；Fast Gate、focused tests、scoped tests 或单条 green command 不等于 Full Review。Campaign 8 可修缺陷和一致性问题，但不得补做缺失的大型 Campaign 6/7 能力；若发现大型缺失，必须退回所属 Campaign。
- Campaign 9：Packaging / EXE 才处理 Packaging / EXE，并需要 real install or run smoke。
- Final Release 只有 Campaigns 1-9 accepted 后才允许开始。
