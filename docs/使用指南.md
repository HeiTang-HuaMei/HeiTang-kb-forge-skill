# 使用指南

## 常用流程

1. 准备本地资料。
2. 使用 `build` 生成 Knowledge Package。
3. 使用 `check-contract` 验证包结构。
4. 使用 `kb-index`、`kb-query` 或 `kb-answer` 做检索和问答。
5. 使用 `generate-documents` 生成 Markdown / DOCX / PDF / PPTX。
6. 根据需要生成 Skill Template、Skill Suite 或 Agent Creation Package。

## Knowledge Package 合同

标准包至少围绕以下文件组织：

- `manifest.json`
- `chunks.jsonl`
- `quality_report.json`
- `ingest_report.md`
- `cards.jsonl`
- `qa_pairs.jsonl`
- `glossary.jsonl`

## 资料输入

Core 支持 Markdown、TXT、DOCX、文本型 PDF、图片/OCR 路由、CSV、TSV、XLSX、HTML、EPUB、ZIP 和多源混合材料。重型 OCR / parser 依赖为可选安装，不作为默认 bundled runtime。

## 安全边界

- 不默认联网。
- 不保存真实 API key、token、cookie、password、secret。
- 不绕登录、验证码、付费墙或平台限制。
- 外部 provider 和网络相关动作必须由用户显式配置。
- 不把 reference-only / planned-not-active 能力写成已接入 runtime。
