# 使用指南

当前使用路径以 v3 PRD 的真实用户流程为准，不按旧技术模块堆叠。

## 1. 启动与配置

1. 创建或选择工作区。
2. 选择本地存储路径。
3. 配置 LLM Provider。
4. 可选配置 Embedding Provider、Redis、向量库、Search Provider 和导出器。
5. 保存配置并执行连接测试。

默认边界：不默认联网，不保存真实 API key，不绕登录；外部 provider 必须显式配置。

## 2. 导入资料到文档库

1. 选择文件、文件夹、网页链接或外部 Skill。
2. 扫描并生成导入任务。
3. 执行类型识别、Parser Provider、可选 OCR 和 chunks 生成。
4. 写入文档库和解析报告。
5. 对失败文件查看原因并按需重试。

文档库不是原始文件夹，而是解析、清洗、结构化后的内容资产库。

## 3. 构建知识库和索引

1. 从文档库选择一个或多个文档。
2. 填写知识库名称、类型和构建配置。
3. 构建知识库。
4. 生成 manifest、chunks、quality report 和 index metadata。
5. 按配置生成关键词索引、向量索引、元数据索引、引用索引或记忆索引。

标准知识包 / OKF 候选层位于文档库与知识库之间。当前可作为标准化方向描述，不作为 runtime 或一级页面使用。

## 4. 检索、RAG 与验证

1. 选择单个或多个知识库。
2. 输入查询。
3. 查看查询改写、命中 chunks、来源 KB、引用证据和重排分数。
4. 生成验证报告。
5. 对冲突或缺漏进行人工纠偏并保存审计。

无 KB 直接回答、无引用回答、多 KB 不标来源均不符合 v3 验收。

## 5. 生成产物

用户目标可以是：

- 生成 PRD、报告、文章、教案、手册等文档。
- 从 KB 或多 KB 生成 Skill。
- 导入外部 Skill 并与本地 KB 融合。
- 创建单 Agent，绑定 KB、Skill、模型、记忆和审计配置。
- 运行 A2A 多 Agent 讨论并导出报告。

历史 Core CLI 仍可生成 Knowledge Package、Document Outputs、Skill Template、Skill Suite 和 Agent Creation Package；这些术语在 v3 中归入编排层产物。

## 6. Core CLI 快速路径

```powershell
python -m pip install -e ".[dev]"
python -m heitang_kb_forge.cli doctor --output .\tmp_doctor
python -m heitang_kb_forge.cli build --input .\examples\quickstart\input --output .\tmp_quickstart_output
python -m heitang_kb_forge.cli check-contract --package .\tmp_quickstart_output --output .\tmp_contract
python -m heitang_kb_forge.cli kb-query --package .\tmp_quickstart_output --query "Summarize the package" --output .\tmp_kb_query
python -m heitang_kb_forge.cli generate-documents --package .\tmp_quickstart_output --output .\tmp_documents
```

Parser/OCR extras 是 dependency-gated，可选安装，不是默认 bundled runtime。

## 7. 历史 Knowledge Package 合同

以下文件名保留给 Core 包合同和旧公共测试追溯，不替代 v3 文档库 / 知识库 / 索引层模型：

- `manifest.json`
- `chunks.jsonl`
- `quality_report.json`
- `ingest_report.md`
- `cards.jsonl`
- `qa_pairs.jsonl`
- `glossary.jsonl`
