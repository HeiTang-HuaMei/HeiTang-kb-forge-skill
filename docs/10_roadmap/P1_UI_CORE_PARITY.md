# P1 UI Core Parity

P1 is the next required product gate before v4.0 can start. It is not complete.

## Goal

Prove that the UI can guide or operate the same main workflows already proven by Core:

- workspace setup
- file selection
- KB build
- query and verification
- document generation
- Agent and Skill creation
- local runtime flows
- storage and memory lifecycle views
- release and gate report review
- provider settings without committing secrets

## Acceptance Boundary

P1 requires a separate UI full-operation acceptance gate. Contract views or minimal bridge wiring are not enough to claim full Workbench operation.

## Current Status

- Core pre-v4 RC readiness is complete.
- UI full-operation remains blocked.
- v4.0 must not start until this gate has real evidence.
