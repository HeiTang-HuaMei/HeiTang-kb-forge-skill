# P2 Productization

P2 starts only after P1 UI Core Parity has evidence. It is future work, not a current v4.0 claim.

## Scope

- packaging and installer polish
- release note discipline
- user-facing troubleshooting and diagnostics refinement
- documentation publication hygiene
- final product acceptance loops
- optional provider UX hardening without committing secrets

## Non-Scope Until Proven

- SaaS
- team permission systems
- platform-hosted user data
- cloud sync
- full external vector database production readiness
- v4.0 release or tag
