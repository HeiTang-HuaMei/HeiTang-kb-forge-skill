## Truthfulness Rules

| Label | rc5 rule |
| --- | --- |
| `enabled_real` | May be used only when a visible entry maps to a real runtime/status path, real artifact path, or a real readiness/validation result. |
| `display_only` | Use for status preview, historical evidence, or non-executing informational surfaces. |
| `disabled_boundary` | Use for safety boundaries, missing credentials, external network opt-in, Computer Use, arbitrary shell, and unauthorized marketplace paths. |
| `pending` / yellow | Must not remain on accepted main capabilities. If the capability is declared in v4.3.0 and has a local Core runtime, rc5 exposes it as a real path. |

## Overclaim Controls

- Preview-only legacy static surfaces are not counted as accepted runtime completion.
- Real runtime acceptance is tied to Core artifacts, bridge request mapping, or explicit disabled/config-gated reasons.
- Reports and fixtures must not include raw secrets or unverified stable release claims.

## Current Status

No rc5 main declared capability remains yellow solely because the UI failed to bind an existing local Core runtime. Safety and credential boundaries remain disabled/config-gated by design.
