# v4.3.0-rc3 UI Usability Repair Report

Date: 2026-06-17

Gate status: local validation passed; commit, push, remote CI, and `v4.3.0-rc3` tag remain required before Owner manual smoke.

Owner decision: `v4.3.0-rc2` manual smoke rejected because the real EXE showed UI usability, status consistency, clickability, and layout reachability defects.

## Scope

This gate repairs rc2 delivery blockers only. It does not add new runtime features, does not enable Computer Use, does not open arbitrary shell execution, does not enable an unauthorized plugin marketplace, does not create a GitHub Release, and does not tag stable `v4.3.0`.

## Implemented Repairs

| Area | rc2 blocker | rc3 repair |
| --- | --- | --- |
| Desktop frame sizing | Fixed preview sizing could crop or make the right side unreliable on smaller Windows viewports. | Desktop frame now fits the real viewport with a smaller minimum width and responsive fallback. |
| 1366x768 / scaled Windows usability | Sidebar and topbar consumed too much space; lower nav entries required fragile scrolling. | Sidebar width and vertical density were reduced; topbar non-critical controls collapse on narrower widths. |
| Main page scroll state | Switching pages could preserve a previous scroll offset and open a new page mid-content. | Each page surface now has an isolated key and a non-persistent scroll controller that resets after page changes. |
| Dashboard and dense grids | Fixed metric/card layouts could overflow or hide status/action content. | Dashboard metrics and output format cards use responsive column counts. |
| Document generation layout | Export/output cards could overlap or appear clipped at smaller widths. | Output format grid now uses responsive 5/3/2/1 column layout and stable row sizing. |
| Status coloring | `display_only` could look like yellow pending/boundary status. | Marker logic separates `enabled_real`, neutral `display_only`, and yellow `disabled_boundary`. |
| Campaign 9 status copy | Settings still surfaced rc2 candidate text after rejection. | Campaign 9 status contract and UI sample now identify `v4.3.0-rc3`. |
