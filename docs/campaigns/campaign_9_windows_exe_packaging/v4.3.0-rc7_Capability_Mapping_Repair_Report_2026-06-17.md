# v4.3.0-rc7 Capability Mapping Repair Report

Status: `v4.3.0-rc7_capability_mapping_product_flow_repaired_real_exe_verified_tagged_pending_owner_retest`

Input failure state: `v4.3.0-rc6_owner_retest_failed_capability_mapping_and_product_flow_not_accepted`

Scope: repair product capability mapping and runtime product flow. This gate did not create a GitHub Release, did not tag stable `v4.3.0`, did not enable Computer Use, did not enable arbitrary shell, and did not open an unauthorized plugin marketplace.

## Repair Summary

rc7 removes the previous "button wall" and debug-command surface from the main product workflow. Core action details remain available only in the folded Developer Diagnostics area.

| Area | rc6 failure | rc7 repair |
| --- | --- | --- |
| Import & Parsing | Import, parsing, OCR, chunking were mixed with generic Core-operation surfaces. | Page owns file/folder import, Owner input import, parser/OCR/chunking, source manifest, parse status, and recovery. |
| Document Library | Imported documents were hard to read and status looked report-like. | Page shows real imported file names, source status, parsing state, persistence, chunk references, and refresh. |
| Knowledge Base | KB build looked like command execution and contained example vector rows. | Page owns build/rebuild/view artifacts and shows real KB counts from `manifest.json`, `chunks.jsonl`, `cards.jsonl`, and `qa_pairs.jsonl`. |
| Retrieval & Verification | Search could look display-only or disconnected. | Page owns real query execution, result path, evidence rows, local citation state, and external opt-in boundary. |
| Document Generation | Markdown output was not presented as a natural product action. | Page owns reading-note/report generation, Markdown output state, and generation history. |
| Skill Factory | Skill generation was described as preview-oriented. | Page owns KB-to-Skill generation, package structure, governance state, validation, and export path. |
| Agent Factory | Agent page previously carried unrelated import/search/KB/Skill actions. | Page now owns Agent creation, KB binding, Skill binding, multi-agent discussion, audit, and safety boundary only. |
| Reports & Audit | Report pages could be confused with Dashboard workflow. | Page remains for read-only evidence, security reports, build reports, CI/smoke/no-secret/overclaim evidence, and diagnostics. |
| Settings | External storage/vector configuration was not clearly separated from main-flow success. | Page owns workspace, local storage, Provider Runtime env-only profile, masked API key, external vector DB configuration state, connection test, and persistence status. |

## Main-Flow Ownership

The primary user path is now:

`Import & Parsing -> Document Library -> Knowledge Base -> Retrieval & Verification -> Document Generation -> Skill Factory -> Agent Factory`

The Agent Factory no longer exposes file import, KB build, search, Markdown generation, or Skill generation. Those actions are available only on their owning pages.
