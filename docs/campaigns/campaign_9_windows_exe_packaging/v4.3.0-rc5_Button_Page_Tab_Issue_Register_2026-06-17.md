## Clickability Verification Rules

Every rc5 `enabled_real` main runtime entry must meet all of:

- a visible page action panel;
- a non-null `CoreBridgeRequest`;
- an allowlisted command in `LocalCoreBridge.defaultAllowedActions`;
- output path contained in the configured workspace;
- Web runtime blocked with `web_local_cli_unsupported`;
- desktop runtime protected by no shell syntax, no shell executable, and no secret environment checks.

All registered rc5 issues are repaired in code. Flutter widget tests verify exact page, button, tab, and runtime action semantics; real EXE launch smoke passes; Owner retest remains required for manual packaged-app acceptance.
