## Goal

`import folder -> document library -> parse/OCR/chunk -> build knowledge base -> retrieval verification -> generate document -> export Markdown -> restart persistence`

## Product Flow Changes

- Dashboard now behaves as a workbench home page, showing source documents, knowledge base, retrieval results, generated/exported documents, recent real tasks, and next step.
- Import and parsing owns source selection, import queue, parser/OCR/chunking, failure recovery, and import history.
- Document library owns imported documents, status, preview, source metadata, parsing artifacts, and file actions.
- Knowledge base owns source selection, KB naming/type/enhancement choices, build action, artifacts, quality, versions, and storage boundary.
- Retrieval and verification owns KB selection, query, evidence snippets, citations, scores, correction, and external verification authorization boundary.
- Document generation owns generation type, output format, citation strategy, preview, Markdown generation, export status, and export history.
