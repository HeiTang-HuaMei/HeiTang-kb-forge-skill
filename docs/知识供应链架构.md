# 知识供应链架构

知识供应链把输入资料转成可验证的知识资产。

## 标准链路

```text
source inventory
-> document understanding
-> chunk / metadata
-> source trace
-> evidence map
-> knowledge package
-> retrieval / verification
-> product outputs
```

检索层包含 deterministic query planning、keyword retrieval、local vector retrieval、hybrid retrieval、rerank、evidence selection 和 knowledge accuracy reports。当前默认路径是本地 deterministic 实现；外部 provider 或外部数据库必须显式配置，不计入默认 runtime。

## 四类产品产物

- `knowledge_package`
- `document_outputs`
- `skill_outputs`
- `agent_creation_package`

## 溯源要求

每个知识块应能回到来源、证据、hash 和生成报告。外部来源、手动证据和视觉证据必须明确区分，不能把预检、手动补材料或 reference-only 误写成真实平台抓取成功。
