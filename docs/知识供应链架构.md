# 知识供应链架构

当前知识供应链以 v3 三基线为准。本文只说明从资料到知识资产、再到产物的落地链路。

## 标准链路

```text
资料导入
  → 解析 / 清洗 / 结构化
  → 文档库
  → 标准知识包 / OKF 候选层
  → 知识库
  → 索引层
  → RAG 与验证层
  → 编排层
  → 文档 / Skill / Agent / A2A
```

简写时统一使用：

```text
文档库 → 知识库 → 索引层 → RAG → 编排层 → 文档/Skill/Agent/A2A
```

## 标准知识包 / OKF 候选层

标准知识包位于文档库和知识库之间，用于把文档库内容封装成可版本化、可流通、可追溯的知识资产。OKF 只写作这一层的候选标准。

当前边界：

- 可讨论标准知识包 manifest、metadata、content files、source references 和 version。
- OKF 不作为一级导航。
- OKF 不作为当前 runtime。
- OKF Bundle 导出和 OKF → 知识库构建属于 P2 或后续验收，不得写成已完成。

## 索引层

索引层把知识库变成可检索、可召回、可追溯的结构。索引类型包括：

- 关键词索引
- 向量索引
- 元数据索引
- 引用索引
- 记忆索引

删除、重建或版本回滚知识库时，索引元数据必须同步更新或明确失效，避免下游继续选择旧索引。

## RAG 与验证层

RAG 输出必须有来源、有引用、有验证、有纠偏。多 KB 检索必须标明来源 KB；无 KB 直接回答、无引用回答和无法保存人工纠偏均属于阻断项。

## 产物与溯源

每个文档、Skill、Agent 或 A2A 产物都应能追溯到：

- 来源文档
- 知识库
- 索引配置
- RAG 命中 chunks
- 模板和 Provider 配置
- 编排记录
- 审计或导出记录

历史术语 `Knowledge Package`、`Document Outputs`、`Skill Outputs` 和 `Agent Creation Package` 继续作为产物层兼容术语，但不得覆盖 v3 主链路。
