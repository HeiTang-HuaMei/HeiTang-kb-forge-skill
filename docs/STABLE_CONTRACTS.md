# Stable Contracts

v2.0 adds `stable-check` to inspect whether the workspace contains the expected knowledge package, Skill package, Agent package, workspace, provider registry, prompt profile registry, and LLM audit files.

`extension_readiness` is a reserved map for future versions. It explicitly marks master Skill learning, derived Skill generation, platform distribution, XHS export, and other later capabilities as `not_enabled` in v2.0.
