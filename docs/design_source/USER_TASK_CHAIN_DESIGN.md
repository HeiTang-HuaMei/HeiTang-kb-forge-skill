# 用户任务链设计源

## 目的

本文件定义 7 个主导航下的用户任务链。实现和测试必须按连续任务链验收，不得只验证单个按钮可点击。

## 任务链总览

```text
创建/选择工作区
-> 导入资料
-> 整理资料
-> 文档库
-> 生成/合并/验证知识库
-> 生成文档 / Skill / Agent
-> 成果管理
-> 操作记录与恢复
```

## 导入资料

主任务：把用户资料加入当前工作区，并让用户知道下一步。

必须支持：

- 文件导入
- 文件夹导入
- 链接导入
- 去重提示
- 解析进度
- 失败原因
- 进入文档库的结果

普通入口只保留文件、文件夹、链接。链接不能真实读取时，必须明确显示不可用或需要配置，不能假装导入成功。

## 知识库

主任务：从文档库生成、合并、查看、验证、导出和删除知识库。

必须支持：

- 选择来源资料生成知识库。
- 生成按钮禁用时说明原因。
- 查看来源文档和片段。
- 删除知识库且不误删来源资料。
- 导出知识库并显示导出位置。
- 重启后恢复知识库状态。
- 多知识库合并创建新知识库。

合并知识库必须是派生操作：

```text
K1 + K2 -> K_MERGED
```

合并后：

- 原知识库不变。
- 新知识库有独立身份。
- 来源关系可追溯。
- 片段来源不丢。
- 删除合并知识库不误删来源知识库。
- 重复合并要提示已存在或生成清楚的新版本。

## Skill

主任务：让用户获得可命名、可查看、可导出、可删除的 Skill 包。

必须清楚区分：

- 从知识库生成 Skill
- 导入 Skill

不得暴露内部层级、内部路径或工程状态。成果页只以 “Skill 包” 展示。

## Agent

主任务：创建助手，绑定知识库和 Skill，在知识边界内完成对话或任务。

必须支持：

- 创建和命名助手。
- 保存和删除助手。
- 绑定知识库。
- 绑定 Skill。
- 查看最近引用来源。
- 知识库外问题默认拒答或提示无依据。
- 显式开启通用知识时，必须标注“非当前知识库依据”。

页面应收敛为：

- 创建/配置助手
- 助手对话
- 工作小组

不得出现多个含义冲突的“创建助手”入口。

## 文档生成

主任务：基于知识库生成用户可保存、打开、导出和删除的文档。

任务链：

```text
选择知识库
-> 输入文档名称
-> 选择文档类型
-> 选择模板
-> 选择输出格式
-> 生成
-> 显示文件名、格式、保存位置
-> 打开 / 导出 / 删除
```

主表达应是：

- 生成文档
- 生成报告
- 生成方案
- 生成手册

不得把“结果导出”作为页面主能力。

## 任务工作台

主任务：查看正在进行、已完成、失败和可重试的用户任务。

必须支持：

- 查看任务进度。
- 查看失败原因。
- 重试失败任务。
- 忽略或清理记录。
- 导出操作历史或支持诊断包。
- 说明导出位置和包含内容。

工作小组任务必须表达为多个助手或角色协作完成任务，不得暴露内部 runtime 或 adapter 词汇。

## 配置

主任务：配置外部增强服务，并验证连接状态。

配置页不能阻断本地基础链路。未配置外部服务时，用户仍应能完成：

- 导入资料
- 整理资料
- 基础文档库
- 基础知识库
- 成果管理

## 反向链路

每个成果必须能反查来源：

- 文档能反查知识库和来源资料。
- Skill 能说明来自知识库生成还是外部导入。
- Agent 回答能反查绑定知识库、引用文档和片段。
- 合并知识库能反查 parent_kbs 和 source_docs。

## 非 happy path 要求

每条通过结论必须包含：

- 正常路径。
- 至少三个扰动动作。
- UI、后台真值、重启后三方一致。

扰动动作包括但不限于：

- 重复点击
- 取消删除
- 删除后重建
- 改名
- 切换语言
- 缩小窗口
- 切页后返回
- 中断后重启
- 外部服务未配置
- 导出路径不可写

## 成功标准

任务链闭合必须满足：

- blocker_count = 0
- ordinary_user_task_chains_passed = true
- no_placeholder_as_real_content = true
- no_internal_artifacts_in_results = true
- owner_visible_ui_tested = true
