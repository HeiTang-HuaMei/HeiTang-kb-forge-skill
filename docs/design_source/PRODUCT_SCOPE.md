# 产品范围设计源

## 目的

本文件定义黑糖 Knowledge Workbench 的产品边界。后续需求、UI、架构和测试如果与本文件冲突，应先更新本文件，再进入编码。

## 产品一句话

黑糖 Knowledge Workbench 是一个本地优先的知识工作台，帮助用户把资料整理成知识库，再基于知识库生成文档、Skill、Agent 和可追溯成果。

## 用户目标

- 导入文件、文件夹或链接资料。
- 整理资料并形成可查看的文档库。
- 从真实来源生成、合并、验证、导出和删除知识库。
- 基于知识库生成文档、报告、方案或手册。
- 从知识库生成 Skill，或导入外部 Skill。
- 创建 Agent，绑定知识库和 Skill，并在知识边界内回答问题。
- 查看成果、操作记录、失败原因和恢复动作。
- 在未配置外部增强服务时，仍可完成本地基础链路。

## 不做什么

- 不把工程报告当作普通用户成果。
- 不把审计证据、测试报告、诊断文件展示在普通成果页。
- 不把 Redis、向量数据库、OCR、外部模型等服务本体打包进应用。
- 不让配置页阻断本地可完成的基础流程。
- 不把内部 runtime、adapter、provider、parser、router、matrix 等实现术语暴露给普通用户。
- 不让 Agent 默认使用未绑定知识库的通用知识回答。

## 一级导航

一级导航固定为：

- 导入资料
- 知识库
- Skill
- Agent
- 文档生成
- 任务工作台
- 配置

新增功能必须落入以上入口之一。不得新增一级导航，除非先更新本设计源并完成 Owner 级确认。

## 普通用户成果类型

普通成果页只允许四类成果：

- 知识库
- 文档
- Skill 包
- Agent 包

工程证据、审计报告、测试报告、支持包、运行日志只能进入操作记录、高级诊断、支持诊断包或元数据详情。

## 配置范围

配置页只展示用户必须配置或测试的外部服务：

- AI 模型接口
- Embedding 接口
- Redis 记忆库
- 向量数据库
- 文档解析能力
- 外部服务连接

文档解析能力的产品表达为：

- 基础解析：已可用
- 高级解析：可选安装 / 已安装
- OCR：可选安装 / 已安装
- 操作：测试文档解析

## 用户可见状态

用户可见状态必须使用普通语言：

- 已可用
- 已连接
- 已配置，待测试
- 未配置
- 测试失败
- 可选，未安装
- 需要处理

禁止把内部枚举、测试状态、能力矩阵状态直接显示给普通用户。

## 进度表达

导入和整理资料中的百分比只表示文档解析进度，不表示从导入到生成知识库的全链路进度。

全链路进展必须用阶段状态表达，例如：

- 已添加资料
- 正在解析
- 已进入文档库
- 可生成知识库
- 已生成知识库

## 成功标准

产品范围闭合必须满足：

- 用户能在不理解工程实现的情况下完成主任务。
- 每个成果都能打开、预览、定位、导出或解释失败原因。
- 每个危险操作都能取消、确认、删除后恢复理解。
- 每个失败状态都有下一步动作。
- 每个主链路都能被真实 running UI 验证。
