# 国际化与文案规范

## 目的

本文件定义中文、英文、状态、按钮、错误和禁用词的表达规则。

## 语言原则

- 中文模式不残留内部英文枚举。
- 英文模式不残留系统 UI 中文。
- 用户导入的文件名和正文可保留原语言。
- 工程词不进入普通 UI。

## 按钮文案

按钮只表达用户动作：

- 添加文件
- 添加文件夹
- 添加链接
- 整理资料
- 生成知识库
- 合并创建新知识库
- 验证知识库
- 生成文档
- 生成 Skill
- 导入 Skill
- 创建助手
- 导出
- 删除
- 重试
- 打开位置

避免：

- 运行主链路
- 导入路径
- 执行 runtime
- 生成 evidence
- provider check

## 状态文案

使用统一状态：

- 已可用
- 已连接
- 已配置，待测试
- 未配置
- 测试失败
- 可选，未安装
- 需要处理

## 错误文案

错误文案必须包含：

- 发生了什么。
- 对用户有什么影响。
- 下一步能做什么。

示例：

```text
文件无法解析。你可以重新选择文件，或跳过该来源继续整理其他资料。
```

## 禁用原因

按钮禁用必须说明原因。

示例：

```text
请先选择至少一个来源资料。
```

不要只禁用按钮。

## 进度文案

百分比只用于解析进度。

全链路使用阶段文案：

- 已添加资料。
- 正在解析。
- 已进入文档库。
- 可生成知识库。
- 已生成知识库。

## 禁用词

普通 UI 避免：

- Provider
- Adapter
- Parser
- Router
- Matrix
- dependency_gated
- needs_secret_config
- ready_for_user_selection
- heitang-rc6-needle
- 0/x
- audit report
- acceptance report
- validation report

## 长文本

长名称必须：

- 不遮挡按钮。
- 不撑坏布局。
- 必要时省略并提供 tooltip。
- 默认窗口可读。

## 双语验收

每次 UI 文案改动至少检查：

- 中文页面。
- 英文页面。
- 默认窗口。
- 长中文名。
- 长英文名。
- 错误态。
- 空状态。
