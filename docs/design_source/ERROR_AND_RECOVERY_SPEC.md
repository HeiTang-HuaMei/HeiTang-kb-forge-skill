# 错误与恢复策略

## 目的

本文件定义错误分类、用户表达和恢复动作。失败不是问题，失败后用户不知道怎么办才是 blocker。

## 错误结构

每个错误应包含：

- error_code
- severity：blocker / major / minor
- user_message
- technical_message 可选
- retryable
- next_actions
- affected_object
- diagnostic_ref

普通 UI 只展示用户能理解的信息。技术细节进入操作记录或诊断包。

## 通用用户表达

推荐：

- 未配置。
- 测试失败。
- 需要处理。
- 当前知识库未找到依据。
- 文件无法解析。
- 导出失败，请重新选择位置。
- 来源缺失，无法继续。
- 操作已取消。

避免：

- dependency_gated
- needs_secret_config
- provider failed
- adapter exception
- parser stack
- 0/x

## 导入错误

场景：

- 文件不存在。
- 文件重复。
- 文件格式不支持。
- 文件损坏。
- 链接无法读取。

恢复动作：

- 重新选择文件。
- 跳过重复项。
- 查看支持格式。
- 重试链接。
- 配置外部连接。

## 解析错误

场景：

- 基础解析失败。
- 高级解析未安装。
- OCR 未安装。
- 文档为空。

恢复动作：

- 重试基础解析。
- 可选安装高级解析。
- 可选安装 OCR。
- 移除来源。
- 导出诊断。

## 知识库错误

场景：

- 无来源。
- 生成失败。
- 合并冲突。
- 合并中断。
- 删除失败。
- 导出失败。

恢复动作：

- 选择来源。
- 重试生成。
- 查看冲突来源。
- 重新合并。
- 打开所在位置。
- 重新选择导出路径。

合并中断后不允许半成品显示为可用。

## 验证 / RAG 错误

场景：

- 未选择知识库。
- 问题为空。
- 无依据。
- 引用来源缺失。

恢复动作：

- 选择知识库。
- 输入问题。
- 补充资料。
- 查看来源片段。
- 重新生成知识库。

无依据不是系统失败，但必须明确显示，不得编造。

## 文档生成错误

场景：

- 未选择知识库。
- 文档名为空。
- 模板不可用。
- 输出格式不可用。
- 保存路径不可写。
- 目标文件被占用。

恢复动作：

- 选择知识库。
- 输入名称。
- 更换模板。
- 更换格式。
- 重新选择保存位置。
- 关闭占用文件后重试。

## Skill 错误

场景：

- 未选择知识库。
- Skill 名为空。
- 导入包无效。
- 指针型 Skill 来源缺失。

恢复动作：

- 选择知识库。
- 输入名称。
- 重新选择 Skill 包。
- 恢复来源或重新生成。

## Agent 错误

场景：

- Agent 未命名。
- 未绑定知识库。
- 未绑定 Skill。
- 问题超出知识库。
- 通用知识未开启。

恢复动作：

- 命名助手。
- 绑定知识库。
- 绑定 Skill。
- 补充资料。
- 显式开启通用知识。

## 配置错误

场景：

- AI 模型接口未配置。
- Embedding 接口未配置。
- Redis 未连接。
- 向量数据库未连接。
- 文档解析增强未安装。

恢复动作：

- 填写配置。
- 测试连接。
- 跳过增强项。
- 使用本地基础流程。

配置错误不得阻断本地导入、整理、基础 KB 和成果管理。

## 操作记录

每个失败动作必须写入操作记录：

- 操作名称。
- 对象。
- 失败原因。
- 是否可重试。
- 下一步。
- 诊断引用。

用户可：

- 重试。
- 忽略。
- 清理。
- 导出支持诊断包。

## 严重级别

blocker：

- 数据误删。
- 编造 KB 外答案。
- UI 显示成功但后台失败。
- 半成品显示可用。
- 无法恢复工作区。

major：

- 失败原因不清楚。
- 导出路径不明。
- 跨页面状态不刷新。
- 长文本影响操作。

minor：

- 非关键文案不够好。
- 轻微视觉不一致。

## 恢复验收

每个错误修复必须验证：

- 错误可复现。
- 用户消息可理解。
- 下一步动作可执行。
- 操作记录存在。
- 重试或忽略有效。
- 重启后状态一致。
