# Owner 复验脚本

## 目的

本脚本用于 Owner 快速确认当前窗口是最新 UI，并抽查关键产品闭环。它不是完整测试报告。

## 复验前

确认：

```text
旧 EXE / 旧进程已关闭。
当前窗口来自 flutter run -d windows 或 fresh review build。
已记录 git HEAD、dirty marker、源码时间、exe/app.so 时间。
```

## 5 分钟路径

1. 打开应用，确认当前工作区名称清楚可见。
2. 进入“导入资料”，添加 UI008 样本资料。
3. 整理资料，确认百分比只表示解析进度。
4. 进入文档库，确认来源、摘要或“暂无”、正文预览或“暂无”可见。
5. 生成知识库，查看来源和片段。
6. 问一个 KB 外问题，确认明确无依据、不编造。
7. 生成一个文档，确认显示保存位置。
8. 进入成果页，确认只看到知识库、文档、Skill 包、Agent 包。
9. 查看操作记录，确认失败可重试或有下一步。
10. 重启应用，确认工作区和成果状态一致。

## 不通过条件

任一情况不通过：

- 当前窗口来源不明。
- 普通 UI 出现工程报告当成果。
- KB 外问题编造答案。
- 删除或合并误删来源资产。
- 按钮无反馈。
- 失败没有下一步。
- 重启后状态不一致。

## 复验结论

