# 安全与权限边界

## 目的

本文件定义本地文件、用户数据、外部服务、密钥和诊断包的安全边界。

## 本地文件边界

默认允许应用操作：

- 当前 workspace。
- 当前 workspace 内的测试数据。
- 用户明确选择的导入文件或导出位置。
- 应用生成的 build 产物，且仅在明确授权清理时。

默认不得操作：

- 源码目录。
- 其他项目目录。
- Owner 真实数据目录。
- 系统目录。
- 未经选择的桌面、下载、Program Files、Windows 目录。

## 删除边界

删除必须满足：

- 目标对象明确。
- UI 有确认。
- 操作记录可追溯。
- 不误删来源资产。
- 不误删 parent KB。
- 不误删其他 workspace。

测试删除必须使用 test marker，例如 `UI008_`。

## 密钥边界

不得写入：

- API key 明文。
- Token。
- Cookie。
- Authorization header。
- 私有连接串明文。

可保存：

- 是否已配置。
- 最近测试状态。
- 脱敏后的 endpoint。
- 脱敏后的配置摘要。

## 外部服务边界

外部服务包括：

- AI 模型接口。
- Embedding 接口。
- Redis。
- 向量数据库。
- OCR / 高级解析。
- 外部连接。

规则：

- 不把服务本体打包进 EXE。
- 默认本地基础链路可用。
- 访问外部服务必须由用户配置或明确触发。
- 失败时可解释、可跳过或可重试。

## 诊断包边界

诊断包可以包含：

- 操作记录。
- 错误码。
- 脱敏配置状态。
- 文件路径摘要。
- 对象 id。
- 测试样本标记。

诊断包不得包含：

- 密钥。
- Cookie。
- Token。
- Authorization header。
- 未经用户确认的完整私密正文。
- Owner 真实敏感数据。

## 日志边界

工程日志可以包含内部术语，但不得泄露密钥。

普通 UI 不应显示工程日志。需要用户查看时，应通过“支持诊断包”或“操作历史”提供摘要。

## 网络边界

默认开发和测试应离线可跑。

需要网络时必须明确：

- 触发入口。
- 目标服务。
- 配置来源。
- 失败降级。
- 是否写入操作记录。

## 安全验收

涉及安全边界的改动必须验证：

- secret scan 无泄漏。
- 删除不会越界。
- 诊断包脱敏。
- 未配置外部服务时本地链路仍可用。
- 外部服务失败不会破坏本地数据。
