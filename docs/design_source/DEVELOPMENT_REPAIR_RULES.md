# 修复开发红线

## 目的

本文件定义在“项目已开发完成但发现大量问题”的阶段，如何修复而不扩大风险。它面向修 bug、补闭环、做 UI closure，不面向从零重写。

## 修复前必须先想清楚

每个修复任务开始前必须回答：

- 用户看到的问题是什么？
- 属于哪个主导航或任务链？
- 是真空、假空、占位、混入、阻断、边界错，还是双语/视觉问题？
- 后台是否已有真实产物？
- 修改应该落在 UI、Controller、Product Service、Runtime，还是测试？
- 如何用 latest running UI 证明 Owner 看到的问题已消失？

不清楚时先定位，不编码。

## 最小改动原则

每次修复只做当前问题所需改动：

- 不顺手重构。
- 不顺手换架构。
- 不顺手清理历史代码。
- 不顺手修改 release / gate / status machine。
- 不把多个无关 blocker 打包进一个大改动。

如果一个问题必须跨 UI 和 runtime，先写清楚数据输入、输出和恢复规则。

## 状态机红线

UI closure 修复默认不得修改：

- `capability_chain_status.json`
- P0 / P1 / P2 gate 状态链
- `current_phase`
- `current_gate`
- `remaining_gates`
- `global_goal_complete`

不得进入 Final Owner Review Gate。
不得回开 P2。
不得声明 production ready、release ready 或 industrial acceptance passed。

## 产品 UI 红线

普通产品 UI 不得出现：

- 工程报告当成果。
- 测试证据当成果。
- 审计报告当普通结果。
- 内部 runtime / adapter / provider / parser / router 词汇。
- 内部枚举状态。
- 假摘要、假正文、假问题、假成果。
- 无反馈按钮。
- 无下一步的失败状态。

普通成果页只允许：

- 知识库
- 文档
- Skill 包
- Agent 包

## 数据安全红线

不得误删：

- 源码。
- 其他工作区。
- 原始资料。
- source_docs。
- parent_kbs。
- 非目标知识库 chunks。
- Owner 真实数据。

删除测试数据必须满足：

- 明确 test marker 或用户明确确认。
- 删除对象属于当前测试工作区。
- 删除后有操作记录或可解释状态。

## 知识边界红线

Agent 默认只基于绑定知识库回答。

知识库外问题必须：

- 明确无依据。
- 不编造。
- 不引用不属于绑定 KB 的来源。

通用知识必须由用户显式开启，并标注“非当前知识库依据”。

## 合并知识库红线

合并必须表达为：

```text
合并创建新知识库
```

不得表达为更新、覆盖或修补原知识库。

合并必须保证：

- 原 KB 不变。
- 新 KB 有独立身份。
- parent_kbs 可追溯。
- source_docs 是并集。
- chunks 去重但不丢来源。
- 冲突进入待核查。
- 删除合并 KB 不误删原 KB。
- 中断后不出现半成品可用状态。

## 配置边界红线

外部服务是增强项，不是本地基础链路前置条件。

未配置 AI、Embedding、Redis、向量库、OCR 时，仍应允许：

- 导入资料。
- 整理资料。
- 基础文档库。
- 基础知识库。
- 成果管理。

配置页只展示用户必须配置或测试的外部服务，不展示底层项目名。

## 测试红线

不得用以下内容单独证明 UI closure 通过：

- 源码扫描。
- widget test。
- controller test。
- runtime test。
- 审计报告。
- 旧 EXE 截图。
- 旧 app.so 行为。

每个通过结论必须包含：

- latest running UI 复验。
- 后台真值对账。
- 重启恢复检查。
- 至少三个扰动动作，除非任务极小且说明原因。

## 修复报告格式

每次修复完成只报告：

- 修了什么用户问题。
- 改了哪些文件。
- 跑了什么验证。
- running UI 是否验证。
- 后台真值是否对账。
- 重启恢复是否验证。
- 仍有什么风险。

不要把未验证项写成通过。

## 何时停止编码

出现以下情况应停止继续扩大改动：

- 当前问题根因不明。
- 运行实例无法证明是最新源码。
- 修复需要改状态机。
- 修复会影响 P0/P1/P2 gate 链。
- 修复可能删除真实用户数据。
- 同一改动开始牵扯多个无关任务链。

停止不是失败，是防止把小洞修成大坑。
