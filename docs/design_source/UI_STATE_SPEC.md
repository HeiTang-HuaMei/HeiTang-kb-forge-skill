# UI 状态规格

## 目的

本文件定义页面必须覆盖的状态。开发 UI 时不得只实现成功态。

## 通用状态

每个主页面必须具备：

- 空状态。
- 加载状态。
- 成功状态。
- 失败状态。
- 禁用态及原因。
- 下一步动作。
- 删除确认。
- 导出成功或失败。
- 重启恢复后的状态。

## 当前工作区显示

每个普通页面必须能让用户知道当前工作区。

推荐表达：

```text
当前工作区：<名称>
状态：已可用 / 需要处理
```

不要只显示路径，也不要把隔离规则表作为页面主体反复展示。

## 导入资料

空状态：

- 提示添加文件、文件夹或链接。

加载状态：

- 显示正在解析。
- 百分比只表示解析进度。
- 显示阶段：已添加资料、正在解析、已进入文档库。

成功状态：

- 显示导入数量、去重数量、失败数量。
- 提供进入文档库 / 继续导入。

失败状态：

- 显示失败原因。
- 提供重试、跳过、查看操作记录。

## 文档库

空状态：

- 当前工作区暂无资料。
- 下一步：导入资料。

成功状态：

- 显示真实来源。
- 显示摘要或“暂无摘要”。
- 显示正文预览或“暂无正文预览”。
- 可选择来源生成知识库。

失败状态：

- 显示解析失败原因。
- 可重试解析或移除来源。

## 知识库

空状态：

- 当前工作区暂无知识库。
- 下一步：选择文档库来源生成知识库。

禁用态：

- 无可用来源时生成按钮禁用，并说明原因。

成功状态：

- 显示名称、来源文档、片段数、状态。
- 可查看来源、片段、验证、导出、删除。

合并状态：

- 文案必须是“合并创建新知识库”。
- 合并后显示 parent_kbs、source_docs、去重数、冲突数。

删除状态：

- 删除前确认。
- 删除后列表更新。
- 不默认选中已删除 KB。

## 知识库验证

空状态：

- 先选择知识库。

输入状态：

- 使用清晰列表多选知识库。
- 不使用内部 chip 表达。
- 不预填测试占位问题。

成功状态：

- 显示回答。
- 显示依据来源。
- 显示发现的问题。
- 显示建议补充资料。
- 可保存、重试、查看来源片段。

无依据状态：

- 明确“当前知识库未找到依据”。
- 不编造答案。

## 文档生成

空状态：

- 先选择知识库。

输入状态：

- 文档名称。
- 文档类型。
- 模板。
- 输出格式。
- 来源显示方式。

成功状态：

- 显示文件名、格式、保存位置。
- 提供打开、导出、删除。

失败状态：

- 显示失败原因。
- 提供修改参数、重试、查看操作记录。

## Skill

空状态：

- 从知识库生成 Skill 或导入 Skill。

成功状态：

- 显示 Skill 名称、来源、依赖模型、状态。
- 可查看、导出、删除。

依赖缺失状态：

- 快照型 Skill 可继续使用。
- 指针型 Skill 必须提示来源缺失。

## Agent

空状态：

- 创建助手。

配置状态：

- 命名助手。
- 绑定知识库。
- 绑定 Skill。
- 通用知识开关默认关闭。

对话成功状态：

- 显示回答和来源。

无依据状态：

- 明确当前知识库无依据。

通用知识状态：

- 必须标注非当前知识库依据。

## 任务工作台

状态：

- 进行中。
- 已完成。
- 失败。
- 可重试。
- 已忽略。

失败记录必须提供：

- 查看原因。
- 重试。
- 忽略。
- 导出支持诊断包。

## 配置

状态：

- 已可用。
- 已连接。
- 已配置，待测试。
- 未配置。
- 测试失败。
- 可选，未安装。
- 需要处理。

未配置外部服务时，不得阻塞本地基础链路。

## 长文本与双语

必须覆盖：

- 长中文名。
- 长英文名。
- 中英文切换。
- 默认窗口尺寸。
- 按钮不截断关键动作。
- 正文、问题、来源可读。
