# Public Surface Policy

HeiTang KB Forge is presented publicly as a local Agent knowledge supply-chain Skill.

## Core Positioning

HeiTang KB Forge turns multi-format materials into standardized, traceable, searchable, auditable, evaluable, and reusable knowledge asset packages.

These packages can be used as upstream assets for:

- RAG retrieval
- Agent-callable Skills
- Agent creation packages
- Q&A Agents
- shopping-guide Agents
- education-tutor Agents
- product-manager Agents
- enterprise knowledge-base Agents

## README Boundary

README should stay focused on:

1. What the project is
2. Quick installation
3. Minimal build command
4. Standard output files
5. Main opt-in capabilities
6. Current boundaries
7. Links to detailed docs

README should not contain full development history or all logical version details.

## Repository Root Boundary

The repository root should not contain:

- temporary test output folders
- local planning scratch folders
- backup files
- generated demo output unless intentionally committed
- duplicated historical docs

Detailed capability history belongs in `docs/CAPABILITY_INDEX.md`.
