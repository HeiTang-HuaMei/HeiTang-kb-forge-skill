# 使用指南

本指南按真实用户路径组织，不按旧技术模块组织。

## 1. 工作区与配置

创建或选择工作区，确认本地存储路径、Provider、Embedding、Redis、向量库、Search 和导出器配置。未配置的外部能力必须显示为不可用或需要配置，不得假成功。

## 2. 导入资料

导入文件、文件夹、网页链接或外部 Skill。导入后应能看到任务状态、失败原因、解析报告和删除历史。删除后刷新或重启不应恢复。

## 3. 文档库

文档库展示解析后的内容资产，而不是原始文件列表。用户应能查看正文、chunks、结构统计、关联知识库和删除影响提示。

## 4. 知识库和索引

从文档库选择一个或多个文档创建 KB。构建后应能查看 manifest、chunks、quality report、index metadata 和版本记录。索引层包括关键词、向量、元数据、引用和记忆索引，按配置启用。

## 5. RAG 与验证

检索前必须选择 KB。单 KB / 多 KB 结果应显示来源、引用、命中 chunks、覆盖率、冲突和验证报告。人工纠偏必须可保存、可审计。

## 6. 编排与产物

编排层按目标生成：

- 文档。
- Skill。
- 单 Agent。
- A2A 多 Agent 协作报告。

每个产物都应可追溯到 KB、索引、RAG 上下文、模板、Provider 和编排记录。

## 7. 审计与导出

产物中心应提供查看、删除、导出的一致入口。审计中心记录导入、解析、KB 构建、索引、RAG、文档、Skill、Agent 和 A2A 的关键操作。
