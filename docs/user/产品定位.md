# 产品定位

HeiTang Knowledge Workbench 是知识资产工作台，不是文件管理器，不是单纯 RAG，不是默认联网 SaaS，也不是任意 shell 或 Computer Use 平台。

## 产品边界

```text
文档库 → 知识库 → 索引层 → RAG → 编排层 → 文档/Skill/Agent/A2A
```

UI 仓库只呈现和操作这条链路，不把历史 rc 页面、临时 demo 或旧 Campaign 报告写成新的产品基线。

## 不是文件管理器

文档库保存的是解析、清洗、结构化后的内容资产。原始文件导入只是入口，不能替代文档库、chunks、解析报告、来源和质量信息。

## 不是单纯 RAG

RAG 层必须绑定 KB 和索引，输出要有引用、证据、验证报告和人工纠偏。无 KB 直接回答不是 v3 主路径。

## 是知识资产工作台

用户目标是从资料生成可复用资产：

- 文档。
- Skill。
- 单 Agent。
- A2A 协作报告。
- 审计和导出记录。

## 安全边界

- 不默认上传用户资料。
- 不保存明文 secret。
- 不绕登录、验证码、付费墙或平台限制。
- 外部 provider、Embedding、Redis、向量库、Search 和导出器均按配置启用。
- 登记项目必须分流为能力 Provider、模板资产、`absorbed_into_architecture`、`rejected_no_architecture_gain` 或 `deferred_with_blocker`；不得长期停留在 `reference_only` / `needs_verification`，也不得写成已接入。
